# Harmonia — design spec extension v0.6: CiPA dynamic-hERG binding kinetics

**Status:** **proposed / implemented in v0.6.0** (2026-06-10) · extends [`spec.md`](../../spec.md) (v0.1)
· advances roadmap **Phase B/C** ("Dynamic hERG + the published optimized kinetics")
· **the data are authoritative published values shipped `unverified`; the kernel
coupling is an explicitly-bounded Tier-C reduction and relaxes no §10 guardrail.**

> *v0.1/Phase-B gave hERG records an optional ad-hoc Langmuir `{kon, koff, trapping}`
> binding model (a placeholder, present only on dofetilide and verapamil). v0.6 sources
> the **real** published CiPA dynamic-hERG binding kinetics — the
> [Li et al. 2017](../../dataset/citations/li-2017.json) IKr-Markov drug-binding model,
> optimized per drug against Milnes-protocol data and validated in
> [Li et al. 2019](../../dataset/citations/li-2019.json) — for the **12 CiPA compounds**
> that have them, straight from the [FDA/CiPA repository](https://github.com/FDA/CiPA).
> It adds them as a first-class, citation-backed dataset field and implements the binding
> kinetics as an opt-in kernel path, without touching the calibrated default classifier.*

---

## 0. Why this, and why now

The single most prominent named gap in the roadmap was *"full CiPA Markov hERG +
published optimized kinetics."* The kinetics exist and are open: the CiPA effort fit a
saturating, voltage-dependent, trapping drug-binding model to the hERG channel for the
subset of compounds with Milnes-protocol dynamic data, and published the optimized
per-drug parameters (`Kmax`, `Ku`, `n`, `halfmax`, `Vhalf-trap`) in the FDA/CiPA
repository. Harmonia's thesis is curated, citation-backed, machine-readable pharmacology;
these parameters are exactly that and belong in the dataset.

What was *blocked* was never the data — it was the temptation to fabricate it. v0.6
sources the real values and is scrupulous about the boundary between **the authoritative
published data** (high-value, durable, certain) and **what this reduced kernel can do
with them** (an honest approximation, clearly labelled).

## 1. Scope

**In scope — the data (the durable contribution).**

- A new optional **`cipa_binding`** block on a `channel_block` IKr record, carrying the
  published CiPA dynamic-hERG parameters: `Kmax` (max binding-rate scale), `Ku`
  (unbinding rate, ms⁻¹), `n` (Hill coefficient of the concentration-dependent binding
  rate), `halfmax` (the nᵗʰ power of the half-maximal concentration, nMⁿ), `Vhalf`
  (trapping half-voltage, mV), and the shared fixed `Kt = 3.5×10⁻⁵ ms⁻¹` trapping rate.
- These for the **12 CiPA compounds with published dynamic fits**: bepridil,
  chlorpromazine, cisapride, diltiazem, dofetilide, mexiletine, ondansetron, quinidine,
  ranolazine, sotalol, terfenadine, verapamil. The other 16 CiPA compounds have **no**
  published dynamic-binding data and keep static Hill block — Harmonia does not
  fabricate kinetics for them.
- Source: the FDA/CiPA repository optimal fits (`hERG_fitting/results/<drug>/pars.txt`),
  the model of [Li et al. 2017](../../dataset/citations/li-2017.json), validated in
  [Li et al. 2019](../../dataset/citations/li-2019.json). Shipped **`unverified`** (§9):
  promotion is a contributor confirming the values against the paper supplement / repo.

**In scope — the model (an honestly-bounded reduction).**

- A **`CiPABinding`** kernel model implementing the *exact* CiPA binding kinetics:
  - binding (to the open channel) at rate `Kmax·Ku·Dⁿ/(Dⁿ + halfmax)`,
  - unbinding at `Ku`,
  - trapping `open-bound → closed-bound` at the fixed `Kt`,
  - voltage-dependent un-trapping `closed-bound → open-bound` at `Kt/(1+exp(−(V−Vhalf)/6.789))`,

  tracked as two drug-bound sub-states (`Obound`, `Cbound`) coupled to the reduced
  kernel's IKr **open gate**. Opt-in via `assess(..., herg_dynamic="cipa")`.

**Out of scope (declared — this is the honesty boundary).**

- **The full 9-state CiPA Markov IKr.** The published parameters were fit to a 6-state
  gating + 3-state drug Markov model for IKr. Harmonia's kernel uses a reduced
  Hodgkin-Huxley IKr gate. v0.6 couples the CiPA *binding* sub-model to that gate — it is
  **a Tier-C reduction of the Markov coupling, not the Markov model itself**. A faithful
  full-Markov IKr (and re-validation of the AP against it) is a separate qualified-model
  effort — a **deliberate scope boundary**, not unfinished v0.6 work. v0.6 ships, and
  honestly labels, the reduced coupling; replacing the reduced gate with the full Markov
  is out of this dataset's scope by design (it would be a different, regulatory-grade model).
- **Feeding the default classifier.** The CiPA binding kinetics equilibrate over *many*
  beats (on-rates ~10⁻⁵ ms⁻¹; the official protocol paces ~1000 beats). The reduced
  kernel runs 3 beats by default. So the CiPA path is a **research/demonstration** surface
  (it shows the trapping phenotype and use-dependence); it is **opt-in and does not change
  the default qNet/ΔAPD90 metric, its thresholds, or any recorded performance number.**
- **Re-deriving potency.** `cipa_binding` does not replace the IC50/Hill `parameters` or
  the `source_values`/`variability` that the headline flip machinery uses; it is an
  additional, parallel description of the *kinetics*.

## 2. The model — CiPA binding on the reduced IKr gate

For a drug at free concentration `D` (nM), with the kernel's IKr activation gate `xr`
and membrane voltage `V`, track two drug-bound fractions of the hERG population —
`ob` (bound while the channel is open-accessible) and `cb` (bound and trapped in the
closed conformation):

```
on   = Kmax · Ku · Dⁿ / (Dⁿ + halfmax)            # binding rate to open channels, ms⁻¹
ktv  = Kt / (1 + exp(−(V − Vhalf) / 6.789))        # voltage-dependent un-trapping, ms⁻¹
d(ob)/dt = on · xr · (1 − ob − cb)  −  Ku · ob  −  Kt · ob  +  ktv · cb
d(cb)/dt = Kt · ob  −  ktv · cb
```

The conducting IKr is scaled by the unbound fraction, `IKr = ḡKr · xr · (1 − ob − cb) ·
(V − EK)` — i.e. the drug-bound fraction `b = ob + cb` is the block. This is the same
"block multiplies the current" coupling the static Hill path and the v0.1 Langmuir path
already use, with `b` now produced by the CiPA binding ODE.

**The trapping phenotype, mechanistically.** At diastolic (negative) `V`, `ktv` is
governed by `Vhalf`: a drug with `Vhalf ≈ 0` (dofetilide, `Vhalf = −1`) has `ktv → 0` at
−85 mV, so bound drug cannot return from `Cbound` to unbind — it is **trapped**, and
block accumulates use-dependently over beats. A drug with strongly negative `Vhalf`
(verapamil, `Vhalf = −97`) keeps `ktv ≈ Kt` even at diastole, so it **washes out** of the
closed channel. The published ordering dofetilide (−1) ≫ terfenadine (−82) > verapamil
(−97) reproduces the experimentally-reported trapping phenotype — the validation that the
*mechanism*, not a tuned number, is doing the work.

## 3. Schema delta (backward-compatible)

One new optional property on a `channel_block` record; every existing record validates
unchanged.

```jsonc
"cipa_binding": {                       // NEW (optional), IKr records only
  "Kmax": 35.1, "Ku": 1.816e-05, "n": 1.08, "halfmax": 216.6, "Vhalf": -1.0,
  "Kt": 3.5e-05,                        // shared fixed trapping rate (Li 2017)
  "model": "CiPA IKr-Markov drug binding (Li 2017), saturating + voltage-dependent trapping",
  "source": "FDA/CiPA hERG_fitting optimal fit",
  "citation": "li-2017", "validation_citation": "li-2019"
}
```

## 4. Guardrails — unchanged

Every v0.1 §10 guardrail holds verbatim.

- **The kernel stays Tier C.** The CiPA binding is coupled to a reduced IKr gate, not the
  qualified Markov model; AP-model records do not change tier, and the default classifier
  is untouched.
- **The data ship `unverified`.** The parameters are transcribed from the FDA/CiPA
  repository (LLM/maintainer-sourced); no human has confirmed them against the source
  inside Harmonia. `harmonia info` counts them as unverified. Promotion follows §9.
- **No verdict, no new claim.** The CiPA path reports the same risk *distribution* /
  diagnostic surfaces as before; it adds a mechanism, not an authority.
- **No fabrication.** Only the 12 compounds with published dynamic data get
  `cipa_binding`; the rest keep static Hill, explicitly.

The tell that this layer has crossed its line: wiring the (slow, un-equilibrated in 3
beats) CiPA path into the headline classifier and quoting a new accuracy. That does not
get built.

## 5. Validation & verification contract (CI gates)

| Guard | What it proves | Mechanism |
| --- | --- | --- |
| **Data present** | all 12 CiPA-dynamic compounds carry a `cipa_binding` block with the five fitted parameters + `Kt`; the other compounds do not | dataset test |
| **Schema** | every record (incl. the new field) validates; `harmonia validate` clean | CI |
| **Zero-drug identity** | at `D = 0` the binding ODE has no block (`ob = cb = 0`), so the CiPA path equals the drug-free AP | unit test |
| **Concentration monotonicity** | higher `D` ⇒ more steady-ish block (larger bound fraction) | unit test |
| **Trapping phenotype** | a trapped blocker (dofetilide, `Vhalf ≈ 0`) retains markedly more diastolic block than a non-trapped one (verapamil, `Vhalf ≪ 0`) at matched setup | unit test |
| **Opt-in non-drift** | the **default** assessment (and every recorded qNet/flip/performance number) is byte-identical with v0.5.x; the CiPA path is reached only via `herg_dynamic="cipa"` | backward-compat test |
| **Dataset reproducibility** | the 12 records regenerate byte-identically from `build_records.py` | CI `git diff --exit-code` |

**Human verification (the gate on `verified`).** The parameters are transcribed from the
FDA/CiPA repository; a contributor promoting them confirms each against the repository's
`pars.txt` and the Li 2017/2019 supplements. LLMs assist but never promote.
