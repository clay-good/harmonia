# Harmonia — design spec extension v0.8.3: drug-reference corroboration + sourced EFTPCs

**Status:** **implemented in v0.8.3** (2026-06-23) · extends
[v0.8.2](v0.8.2-pending-human-review.md) (the `pending_human_review` state) ·
**adds no modeling and changes no risk number. It does NOT promote any record to
`verified`.**

> *v0.8.2 introduced `pending_human_review` and filled it for channel-block
> records (the IC50 data). v0.8.3 extends the same corroboration mechanism to the
> 28 **drug_reference** records — the CiPA expert risk label + free EFTPC — by
> sourcing and verifying the published CiPA exposures and categories. The
> dataset's `pending_human_review` count rises from 56/104 to **83/104**.*

---

## 1. What was corroborated

Each `drug_reference` record carries the CiPA **expert risk label**
(high/intermediate/low) and the **free EFTPC** (effective free therapeutic plasma
concentration, nM) used as the simulation exposure. Both are now checked against
an independent published CiPA source, in a new reference table
`dataset/references/cipa_drugref_reference.json`:

- **Training drugs (12)** — FDA/CiPA `AP_simulation/data/CiPA_training_drugs.csv`
  (`therapeutic` = free EFTPC nM; `CiPA` = 2/1/0). Independent of the `li-2017`
  primary citation on those records. **Verified value-by-value** against the file:
  all 12 categories match; 11/12 EFTPCs match exactly (terfenadine 9 vs 4 nM,
  2.25×).
- **Validation drugs (16)** — FDA/CiPA `AP_simulation/data/newCiPA.csv` (the
  simulation input behind Li et al. 2019), **verified against the raw file**, with
  EFTPCs **cross-corroborated** against Llopis-Lorente 2022 Table 1 and categories
  also confirmed in Colatsky 2016. (The CSV's `freedrug`/`freedrugStrauss` columns
  are internally inconsistent — off by 1000× for metoprolol/risperidone — so the
  `therapeutic` column is used, as the FDA `AP_simulation/README` specifies.)

## 2. The promotion rule (drug_reference)

A `drug_reference` record is promoted `unverified → pending_human_review` iff
**both**: the CiPA risk **category matches exactly**, AND the **EFTPC agrees
within 3×** (a looser fold than the IC50 5×, because EFTPC reflects a single
exposure value that varies less but is label-dependent). The `extraction.corroboration`
block records the `eftpc_fold_diff`, `category_match: true`, source, and citation.
This never changes a value — provenance only.

**Result:** 27/28 drug-reference records promote. Combined with the 56 channel-block
records from v0.8.2, **83/104 records are now `pending_human_review`**; `verified`
remains **0/104** (§9).

## 3. What it surfaced — the ibutilide EFTPC discrepancy

`drug_reference.ibutilide` did **not** corroborate: its free EFTPC is **0.52 nM**
in Harmonia, but **100 nM** in both independent CiPA sources (FDA `newCiPA.csv`
and Llopis-Lorente 2022) — a ~**200× discrepancy**. As designed, the record stays
`unverified` (the disagreement is surfaced, not silently filled). ibutilide is a
high-risk drug; a 200×-too-low exposure would make the kernel under-read its risk,
so this is a genuine data-quality flag a human should reconcile against the
primary source (analogous to the cisapride/terfenadine errors the v0.8 cross-check
caught, fixed in v0.8.1). It is reported, not auto-corrected.

## 4. The honesty contract (unchanged from v0.8.2)

`pending_human_review` ≠ `verified`; no value is invented (every reference number
is verified against its source file or cross-corroborated); corroboration is a
gate, not a rewrite; disagreement stays `unverified` and visible. AP-model records
(reduced kernel, not yet cross-checked against the canonical CellML) and
population records (hypothesis-tier, illustrative magnitudes) legitimately remain
`unverified` — they are not transcribed measurements with an independent published
counterpart to corroborate.

## 5. Non-drift

One reference file, two schema fields (`eftpc_fold_diff`, `category_match` on the
existing `corroboration` block), one new citation source text, and review-status
bookkeeping. No risk number, threshold, classification, or simulation output
changes; record diffs are provenance-only.
