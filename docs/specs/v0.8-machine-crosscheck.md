# Harmonia — design spec extension v0.8: machine cross-check against the published CiPA reference

**Status:** **implemented in v0.8.0** (2026-06-23) · extends [`spec.md`](../../spec.md) (v0.1)
· hardens the **provenance/trust surface** (§9) · **adds no modeling, changes no
risk number, and — critically — does NOT promote any record to `verified`.**

> *Harmonia's honesty posture (spec.md §9) ships every record as
> `review_status: "unverified"`: the values are literature-derived, but nobody has
> opened the source PDF inside Harmonia, and an LLM may never promote a record to
> `verified`. That is correct, but it left a reader with **no automated signal at
> all** of whether a transcribed IC50 is even in the right ballpark — the 0/104
> verified count is honest yet uninformative. v0.8 adds a second, independent
> rendering of the canonical CiPA block table and diffs every channel-block record
> against it. The result is a new, **honestly weaker** provenance signal —
> `machine_cross_checked` — that sits between "unverified" and "verified": it
> confirms agreement with a published number, never that a human read the source.*

---

## 0. Why this, and why now

The single biggest gap between "credible demonstrator" and "resource a safety
scientist would cite" is that **0 of 104 records are human-verified**. Closing
that fully is curatorial labor reserved for human contributors (spec.md §9). But
a large fraction of that risk — a transcribed IC50 that is silently off by a
factor of 10 or 1000 (a unit slip, a dropped exponent, a mis-keyed mantissa) — is
mechanically detectable *today*, without a human, by comparing against an
independent published copy of the same number.

That is exactly what the CiPA program provides: the 12 training drugs have a
canonical, peer-reviewed multichannel block table (Li et al. 2017), published in
machine-readable form in the FDA/CiPA reference implementation. Harmonia's records
were transcribed by the maintainer from the primary literature; the reference
table is a **different transcription pass of the same published values**. Agreement
between the two is real evidence the record is faithful; a large divergence flags
the record for human review.

v0.8 builds that comparison and surfaces it. It found real defects on first run
(§4), which is the point.

## 1. Scope

**In scope**

- A published-reference table, `dataset/references/cipa_block_reference.json`:
  IC50 (nM) and Hill per drug × current for the 12 CiPA training drugs,
  transcribed from **Li et al. 2017** (the `li-2017` citation already in the
  dataset) via the FDA/CiPA machine-readable file, with full provenance.
- A deterministic build tool, `dataset/tools/build_cipa_reference.py`, mirroring
  `build_records.py`: the committed JSON must be byte-identical to a fresh run
  (CI gate).
- `harmonia.cross_check(ds, drug=None)` → `CrossCheckReport`: per-record
  IC50/Hill fold-difference, a four-way status, and a computed
  `machine_cross_checked` flag.
- CLI `harmonia crosscheck [drug] [--strict]` and a one-line summary in
  `harmonia info`.

**Explicitly NOT in scope**

- **Promotion to `verified`.** A machine cross-check is not a human reading the
  PDF. The `verified` count stays driven solely by `extraction.verified_by`
  (spec.md §9). `machine_cross_checked` is a *separate, computed, weaker* signal.
- **Auto-correcting records.** The cross-check *flags* divergences; it never edits
  a curated value. Correcting a flagged record is a human curation decision that
  must trace to the primary source.
- **Max-block / dynamic-binding cross-check.** The Li 2017 summary table carries
  IC50 and Hill only. Cross-checking `max_block_observed_percent` (against the raw
  Crumb 2016 dose-response) and the `cipa_binding` kinetics (against the FDA/CiPA
  `hERG_fitting` optimal fits) are natural follow-ons, noted here as future work.
- **The 16 validation drugs.** They are absent from the training-drug table and
  cross-check to `no_reference` (reported honestly), pending a Li 2019 reference.

## 2. The reference table

`build_cipa_reference.py` embeds the Li 2017 multichannel IC50/Hill table as
auditable facts and normalizes it to one entry per drug × channel (the CiPA
current name `hERG` maps to Harmonia's `IKr`). 65 entries across 12 drugs.

**Provenance & licensing.** The numeric values originate in Li et al. 2017
(DOI 10.1161/CIRCEP.116.004628). They were obtained in machine-readable form from
`github.com/FDA/CiPA`, `Hill_fitting/data/Li2017_IC50.csv` @ commit `e881df6`.
The FDA/CiPA *software* repository is GPL-3.0; the IC50/Hill *values* reproduced
here are uncopyrightable factual measurements from the primary publication,
redistributed under Harmonia's CC-BY-4.0 dataset license **with attribution to
Li 2017**. No GPL-licensed file is vendored — only the facts, with the FDA/CiPA
repository credited as the machine-readable conduit.

## 3. The comparison

For each channel-block record with a matching reference entry, the cross-check
computes the IC50 fold-difference `f = max(rec/ref, ref/rec)` and classifies:

| Status | Condition | Meaning |
| --- | --- | --- |
| `match` | `f ≤ 2` | strong agreement |
| `minor` | `2 < f ≤ 5` | within the documented inter-lab spread — not an error |
| `divergent` | `f > 5` | **suspicious** — likely a unit/transcription error or mis-mapped drug/channel; flag for human review |
| `no_reference` | no published value | drug/current outside the 12-training-drug table |

`machine_cross_checked := status ∈ {match, minor}`.

**Why 5× is the divergence threshold, not something tighter.** The project's
entire thesis is that IC50s vary several-fold across labs, and a record's central
value is a geometric mean that may legitimately differ from any single published
value. A 2–3× gap is *expected*, not a defect. The 5× line deliberately mirrors
the dataset's own `fold_range > 5` failure-mode trigger (spec.md): only an
order-of-magnitude-ish discrepancy is treated as suspicious. The Hill coefficient
is reported as a secondary annotation (`|Δh| > 0.3`) and never gates the status —
Hill is far less reproducible across platforms than IC50.

## 4. What it found on first run (the validation of the idea)

Run over all 68 channel-block records: 38 have a published reference, 36 are
`machine_cross_checked` (32 `match`, 4 `minor`), and **2 are `divergent`** —

- **`channel_block.cisapride.ical`** — recorded **9 258 nM**, published
  **9 258 075 nM**: identical mantissa, off by exactly **1000×**. Cisapride is a
  negligible L-type-calcium blocker; the recorded value is three orders of
  magnitude too potent — a textbook dropped-exponent / unit-scale transcription
  error.
- **`channel_block.terfenadine.inal`** — recorded **2 000 nM**, published
  **20 056 nM**: ~**10×**.

Neither was auto-corrected. Both were surfaced for a human to reconcile against the
primary source — which is precisely the workflow §9 mandates. The other 32
training-drug matches (e.g. dofetilide IKr 5.06 vs 4.87 nM, ×1.04; verapamil IKr
252 vs 288 nM, ×1.14) are independent evidence the bulk of the transcription is
faithful.

**Update (v0.8.1) — reconciled.** Both flagged channels were reviewed against the
raw Crumb-2016 dose-response (the cited source), which shows ≤2.5% (cisapride
ICaL, at the 125 nM top dose) and ~15% (terfenadine INaL, at 800 nM) block — i.e.
the IC50s are *unidentifiable* (max block << 60%). Per the reliability gate both
are now **Tier D** with the unidentifiable failure mode (the IC50 point estimate
is the Li-2017 extrapolated fit, kept for provenance only), exactly as
`ranolazine.ical` already was. The committed dataset now has **zero** cross-check
divergences, guarded by a test. Honest cost: correcting `terfenadine.inal`
(faithfully weaker protective late-Na block) pushed the reduced kernel to
over-predict terfenadine to `high` (an adjacent error), so qNet exact accuracy
moved training 10/12 → 9/12 and all-28 17/28 → 16/28, within-one-category
unchanged at 100%. This is the cross-check working end to end: flag → reconcile to
source → a more faithful dataset, with the accuracy cost reported, not hidden.

## 5. The honesty contract (how this stays inside §9)

1. **`machine_cross_checked` ≠ `verified`.** They are different fields with
   different meanings, surfaced separately. `harmonia info` prints both, on
   separate lines, and the cross-check summary states in full that it "is NOT
   human `verified`." The `verified` count remains 0/104.
2. **No record is mutated.** The reference is a separate file; the cross-check is
   a derived, on-demand computation (like `harmonia performance`), never baked
   into records. The "never hand-edit a record" guarantee is untouched.
3. **A pass is not a safety claim.** A `match` says only that two independent
   renderings of one published number agree. It says nothing about whether that
   number is the right input for a given patient, exposure, or assay — and the
   tool still emits no verdict.
4. **Coverage is reported, never hidden.** `no_reference` is a first-class status;
   the summary always prints how many records had no published value to check
   against, so a reader never mistakes "not checked" for "checked and fine."

## 6. Non-drift

This extension sources a reference file, adds one module, one CLI verb, one
public function, and one `info` line. It changes **no** risk number, **no**
calibrated threshold, **no** record, and **no** previously-reported output. Every
prior test passes unchanged; the new behavior is purely additive.
